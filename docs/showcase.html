Features

Everything in one tool

Profile isolation, smart failover chains, and seamless context handoff — all from a single CLI.

🔑

Multiple OAuth accounts

Each profile is an isolated config directory (~/.claude-<name>/) with its own OAuth login via CLAUDE_CONFIG_DIR, while sharing settings, hooks, and plugins back to your main ~/.claude/ via symlinks.

⛓

Fallback chains

Define an ordered list of profiles. When the active one is unavailable, the next healthy one is tried automatically. Chains are created with a single command and can be managed live.

⚡

Smart automatic failover

Failover triggers on exactly the right errors: usage/rate limits (429, quota exceeded), server errors (5xx, overloaded), and expired auth. A generic crash surfaces immediately — it is never silently swallowed.

⏱

Per-profile cooldowns

Rate-limit cooldowns respect the server's reset time (or default to 1 hour). Server errors cool down for 2 minutes. Auth failures flag the profile as "needs auth" until you re-login. Health state persisted in state.json.

🔄

Cross-session context handoff

Stop/SessionEnd hooks snapshot conversation context to a shared handoff directory. The next session's SessionStart hook injects that context automatically — so the new account picks up where the last left off.

🖥

Interactive TUI by default

The full Claude Code TUI is the default run mode, not headless. A supervisor process re-launches the next healthy profile when one exits after a limit. Headless mode with auto-retry is also fully supported via --headless.

Quickstart

Up in under a minute

Install once, create your profiles, log in, and run.

1

Install and initialise

shell 
$ npm install -g claude-profiles
$ claude-profiles init
2

Create and authenticate profiles

shell 
$ claude-profiles profile create work
$ claude-profiles profile create personal
$ claude-profiles profile login work
$ claude-profiles profile login personal
3

Create a fallback chain

shell 
$ claude-profiles chain create default --profiles work,personal
4

Run with automatic failover

shell 
$ claude-default                              # interactive TUI, first healthy account
$ claude-default -- -p "summarize this repo"   # headless with auto-failover

Profile commands

Command Options Description
profile create <name> -y, --yes
--shell <file>
--share-claude-md / --no-share-claude-md
--share-statusline / --no-share-statusline
--description <text>

--chain <name> 		Create a new isolated profile directory under ~/.claude-<name>/ with shared symlinks back to ~/.claude/. --priority: lower number = tried first in a chain

Chain commands

Command Options Description
chain create <name> --profiles <a,b,c>
--shell <file>
--no-alias 		Create a named fallback chain from an ordered list of profiles. By default, creates a shell alias (e.g. claude-default) for the chain.
chain list — List all chains and their member profiles.
chain add <name> <profile> — Append a profile to an existing chain.
chain remove <name> <profile> — Remove a profile from a chain.
chain delete <name> — Delete a chain entirely (profiles are not deleted).
chain status — Show each profile's current state: healthy, cooling down (with time remaining), or needs auth.
chain reset [profile] — Clear cooldowns and needs-auth flags. Omit profile to reset all.

Run command

Command Options Description
run [options] -- <claude args> -c, --chain <name>
-p, --profile <name>
--interactive
--headless
--new 		Run Claude via a chain or a single profile. Everything after -- is forwarded to the claude binary. Mode auto-detect: if -p / --print appears in claude args, runs headless. Otherwise interactive (TUI) is the default. --new: start a fresh continuity thread, ignoring any previous handoff context.

Handoff (context continuity)

Command Description
handoff status Show the current handoff context state for all chains.
handoff enable Install continuity hooks into ~/.claude/settings.json (Stop/SessionEnd + SessionStart hooks).
handoff disable Remove continuity hooks from ~/.claude/settings.json.
handoff clear [chain] Discard saved handoff context. Optionally scoped to a single chain.

How it works

Failover walkthrough

When an account hits a limit, the next healthy profile takes over — with the conversation context already loaded.

🚀

Session starts on work account

claude-default launches the Claude TUI using the first healthy profile in the chain — work. Normal conversation proceeds.

work · active
⚠️

work hits its usage limit

Claude returns a usage-limit error (HTTP 429 or "usage limit reached"). This is a recognised failover trigger. A generic crash would surface immediately and would not trigger failover.

429 · usage limit reached
💾

Context snapshot & cooldown recorded

A Stop/SessionEnd hook fires and snapshots the conversation summary to ~/.claude/.claude-profiles/handoff/default/current.json. The work profile is marked cooling down for ~1 hour (or until the server-supplied reset time).

work · cooling down ~1h
🔄

Supervisor relaunches with personal

The supervisor sees work is on cooldown and picks the next healthy profile — personal. A new Claude TUI session is started under that account.

personal · healthy
✅

Context injected — conversation continues

personal's SessionStart hook reads the handoff file and injects the prior conversation summary via additionalContext. The new session resumes seamlessly. Continuity is only activated after a failover — a clean session ends fresh.

context restored · continuing
$ claude-profiles chain status
chain  default
  [1]  work        cooling down   (47m remaining)
  [2]  personal    healthy
💡

Interactive failover = relaunch with context, not a transparent mid-session swap. The Claude TUI is a full terminal application — it cannot be swapped while a conversation is live. Failover means the supervisor closes the current session and opens a new one under the next healthy profile, with the prior context already injected. Headless (--headless) mode can retry within the same invocation.

Configuration

Environment variables

Variables set by the supervisor and read by hooks. Most are managed automatically.

Variable Description
CLAUDE_CONFIG_DIR Points to the active profile's isolated config directory (~/.claude-<name>/). Set automatically per profile to keep OAuth credentials isolated. Can also be set manually to override.
CLAUDE_PROFILES_CLAUDE_BIN Override the path to the claude binary. Useful if you have multiple Claude Code versions installed or a custom wrapper.
CLAUDE_PROFILES_CHAIN internal Set by the supervisor when launching a session. Read by hooks to know which chain is active. Not intended for manual use.
CLAUDE_PROFILES_THREAD internal Set by the supervisor to identify the current continuity thread. Read by SessionStart and Stop hooks. Not intended for manual use.
